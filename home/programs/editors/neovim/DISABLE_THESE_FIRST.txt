# Disable These Plugins First (Most Likely to Cause Hangs)

## Quick Disable Commands

Run these from the neovim directory:

```bash
cd /Users/wm/nix-config/home/programs/editors/neovim

# Disable high-risk plugins
./disable-plugin.sh disable treesitter
./disable-plugin.sh disable conform
./disable-plugin.sh disable copilot
./disable-plugin.sh disable avante
./disable-plugin.sh disable lazygit
./disable-plugin.sh disable fzf-lua

# Check status
./disable-plugin.sh status

# Test nvim
nvim
```

## If That Works, Enable One by One

```bash
# Enable and test each
./disable-plugin.sh enable lsp
nvim  # Test

./disable-plugin.sh enable mason
nvim  # Test

./disable-plugin.sh enable blink-cmp
nvim  # Test

# Continue with others...
```

## Nuclear Option - Disable ALL Non-Essential

```bash
./disable-plugin.sh disable-all
nvim  # Should work with minimal config
```

## Top Suspects (In Order)

1. **treesitter.lua** - Parser installation hangs
2. **conform.lua** - Auto-format on BufLeave
3. **copilot.lua** - Network/auth required
4. **avante.lua** - AI plugin, network required
5. **fzf-lua.lua** - Can hang on large directories
6. **lazygit.lua** - Requires lazygit binary
7. **neo-tree.lua** - File tree can be slow
8. **dap.lua** - Debugger setup
9. **snacks.lua** - Dashboard/picker can hang

## Essential Plugins (Keep Enabled)

- colorscheme.lua
- lsp.lua
- mason.lua
- which-key.lua
- noice.lua (maybe)
- lualine.lua (maybe)

## After Finding the Culprit

Once you identify which plugin causes the hang, you can:

1. Keep it disabled
2. Fix the configuration (I can help)
3. Replace with alternative plugin
